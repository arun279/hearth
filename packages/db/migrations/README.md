# D1 migrations

Generated migrations land here via `pnpm db:generate`. Most files are auto-generated by drizzle-kit from the schema in `packages/db/src/schema/`; do not hand-edit them.

## Hand-written migrations

A small number of migrations are intentionally hand-written and excluded from drizzle-kit's diffing path. They sit alongside the generated files and run in the same numeric sequence. Drizzle's snapshot files in `meta/` reflect only the generated migrations — that's the divergence the `check:table-inventory` gate enforces.

The hand-written set covers SQLite features drizzle-kit cannot manage cleanly:

- **`0001_library_fts5.sql`** — creates the FTS5 virtual table `library_items_fts` and the `AFTER INSERT / DELETE / UPDATE` triggers that mirror `library_items` into it. Drizzle would emit `DROP TABLE library_items_fts` on every diff because virtual tables aren't part of its diffable surface.
- **`0002_instance_settings_seed.sql`** — seeds the singleton `instance_settings` row.
- **`0008_library_fts5_tags.sql`** — extends `library_items_fts` with a `tags` column whose triggers extract values from `library_items.tags_json` via `json_each`. Recreates the FTS table because FTS5 has no `ALTER TABLE ADD COLUMN` path.

Never re-emit any of these via `pnpm db:generate` and never let drizzle-kit "regenerate" the FTS table — both paths drop content and re-create empty indexes.

## Order

Migrations apply in lexicographic order by filename. The `meta/_journal.json` file lists the drizzle-tracked entries; the hand-written files run between them by virtue of their numeric prefix.
